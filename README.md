# Code Pipeline 数据模型设计

本文档记录了 `code-pipeline` 系统中“流水线配置”的数据模型设计。

## 数据模型结构

### 1. 流水线 (Pipeline)
流水线是系统的核心管理单元。每个流水线包含以下属性：

| 属性名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| **ID** | Integer / String | 唯一标识符（自增 ID 或 UUID） |
| **流水线ID** | String | 业务层面的流水线标识 ID |
| **名称** | String | 流水线名称 |
| **类型** | String / Enum | 流水线触发或运行类型，例如：`MR` (Merge Request 触发)、`每日构建` (Daily Build) 等 |
| **组名称** | String | 归属的组名称 |
| **描述** | String | 流水线的详细描述信息 |

---

### 2. 执行方案 (Execution Plan)
每个流水线包含一系列的“执行方案”，用于定义具体的执行逻辑与配置：

| 属性名称 | 类型 | 描述 |
| :--- | :--- | :--- |
| **代码仓** | String | 代码仓库地址 (Repository URL) |
| **分支** | String | 代码分支 (Branch) |
| **用户名** | String | 用于访问仓库的用户名 |
| **密码** | String | 用于访问仓库的密码（应加密存储） |
| **自定义属性** | JSON | 自定义配置属性，采用 JSON 格式存储，以便于扩展 |

## 关系说明
* **一对多关系**：一条“流水线”可以关联多条“执行方案”，形成一对多的关系。
